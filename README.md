# AI Endpoint Visibility Sensors for Tanium

Example Tanium sensor logic for identifying endpoint indicators of AI tool usage across Windows and macOS systems.

## Purpose

This project supports defensive AI governance by helping security teams identify:

- Local AI binaries and LLM runners
- AI/ML developer frameworks and libraries
- AI-related browser extensions

The goal is visibility, not blocking. These sensors help organizations understand where AI tools may exist across endpoints.

## Alignment

This project is conceptually aligned with:

- NIST AI Risk Management Framework
- AI governance
- Endpoint visibility
- Cybersecurity monitoring
- Responsible AI adoption

## Sensors

| Sensor | Purpose |
|---|---|
| Local AI Binaries and LLM Runners | Detects local AI apps and model runners |
| AI Developer Frameworks and ML Libraries | Detects ML/AI Python libraries and environments |
| AI Browser Extensions | Detects AI-related Chrome and Edge extensions |

## Disclaimer

This repository is for educational and defensive security purposes only. It does not contain production data, customer data, internal Tanium details, credentials, hostnames, or real endpoint results.
